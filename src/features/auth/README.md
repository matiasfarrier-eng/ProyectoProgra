Feature `auth`

Estructura esperada:
- `api/` - endpoints y adaptadores HTTP
- `components/` - componentes UI específicos
- `hooks/` - hooks personalizados (p. ej. `useAuth`)
- `pages/` - páginas del módulo
- `services/` - llamadas a la API y lógica de negocio
- `store/` - estado local o global del feature
- `types/`, `schemas/`, `utils/`, `queries/`

Export público: importa desde `@features/auth`.
